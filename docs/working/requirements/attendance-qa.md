# 勤怠管理アプリ — 要求仕様 Q&A

## ユーザーストーリー（たたき台）

### 打刻
- 社員として、出勤時に打刻したい。勤務開始時刻を記録するために。
- 社員として、退勤時に打刻したい。勤務終了時刻を記録するために。

### 勤怠一覧
- 社員として、自分の月次勤怠一覧を確認したい。勤務実績を振り返るために。

### 管理
- 管理者として、社員の勤怠一覧を確認したい。勤務状況を把握するために。

---

## 仕様を確定するための質問

### 1. ユーザー・認証

[Question] 利用者の種類は「一般社員」と「管理者」の2種類ですか？他にロール（部門長、人事など）はありますか？
[Answer]一般社員、部門長、管理者の三種類

[Question] 認証方式は何を想定していますか？（ID/パスワード認証、SSO、認証なし（ワークショップ用に簡略化）など）
[Answer]ID/パスワード認証

[Question] 社員マスタ（社員の登録・編集・削除）は本アプリのスコープに含みますか？
[Answer]全て含む

### 2. 打刻

[Question] 打刻の種類は「出勤」「退勤」の2種類ですか？（休憩開始/終了、外出/戻りなども必要ですか？）
[Answer]出勤・退勤のみ

[Question] 1日に複数回の出退勤（中抜け）を許可しますか？それとも1日1回ずつ？
[Answer]許可

[Question] 打刻の修正（打刻忘れ、誤打刻の訂正）機能は必要ですか？必要な場合、承認フローは要りますか？
[Answer]修正あり　承認フローなし

[Question] 打刻時刻は「現在時刻を自動取得」のみですか？それとも手動で時刻入力も可能ですか？
[Answer]手動可能

### 3. 勤怠データ

[Question] 勤務時間の計算ルールはどうしますか？（例: 休憩1時間を自動控除、深夜割増の計算、残業の閾値は8時間 etc.）
[Answer]休憩1時間控除あり　定時を7時間15分　残業はx1.25倍計算

[Question] 休日・祝日の扱いはどうしますか？（カレンダーマスタが必要？土日固定？）
[Answer]カレンダーマスタあり

[Question] 有給休暇・各種休暇の管理は本アプリのスコープに含みますか？
[Answer]含む　有給は一律20日

### 4. 出力・レポート

[Question] 月次の勤怠レポート（CSV/PDF出力など）は必要ですか？
[Answer]必要

[Question] 残業時間のアラート（例: 月45時間超過で警告）は必要ですか？
[Answer]必要 30時間

### 5. 技術・非機能要件

[Question] 想定する利用者数の規模感は？（〜10人の小規模、〜100人の中規模、それ以上）
[Answer]中規模

[Question] モバイル対応は必要ですか？（レスポンシブ Web、ネイティブアプリ、PC のみ）
[Answer]PCのみ

[Question] データの保持期間に要件はありますか？（例: 3年保存）
[Answer]1年

### 6. スコープ

[Question] 初回リリース（MVP）に含めたい最小限の機能はどこまでですか？
[Answer]4の出力・レポート以外は必須

# Unit of Work — 作業分割

## 分割方針

- **2人で並行実装**できるように Phase を設計
- 共通基盤（Unit 0）を先に完成させてからドメイン Unit を並列着手
- Backend と Frontend を同一 Unit 内に含め、縦割り（フィーチャー単位）で分割

## Unit 一覧

| Unit | 名称 | 担当 | 依存 |
|------|------|------|------|
| Unit 0 | 共通基盤 | 2人共同 | なし |
| Unit 1 | 認証・社員管理 | 担当A | Unit 0 |
| Unit 2 | 打刻・勤怠一覧 | 担当B | Unit 0 |
| Unit 3 | 有給休暇 | 担当A | Unit 1 |
| Unit 4 | カレンダー・残業アラート | 担当B | Unit 2 |

## 依存図・Phase

```
Phase A: ┌──────────┐
         │  Unit 0  │  共通基盤（2人で共同）
         │  基盤    │
         └────┬─────┘
              │
Phase B: ┌────┴─────┐─────────────┐
         │          │             │
    ┌────▼────┐  ┌──▼───────┐    │
    │ Unit 1  │  │  Unit 2  │    │  ← 2人で並列
    │ 認証    │  │  打刻    │    │
    │ 社員管理│  │  勤怠一覧│    │
    └────┬────┘  └──┬───────┘    │
         │          │             │
Phase C: │          │             │
    ┌────▼────┐  ┌──▼───────────┐│
    │ Unit 3  │  │  Unit 4      ││  ← 2人で並列
    │ 有給休暇│  │  カレンダー  ││
    │         │  │  残業アラート ││
    └─────────┘  └──────────────┘│
```

## Phase 計画

| Phase | 内容 | 並列度 | 目安 |
|-------|------|--------|------|
| A | Unit 0: プロジェクト骨格・DB・認証基盤・テスト環境 | 2人共同 | 最初 |
| B | Unit 1 + Unit 2 を並列実装 | 2人並列 | Unit 0 完了後 |
| C | Unit 3 + Unit 4 を並列実装 | 2人並列 | Phase B 完了後 |
| 最後 | 統合テスト・E2E | 2人共同 | Phase C 完了後 |

## 担当割り当て案

| 担当 | Phase B | Phase C |
|------|---------|---------|
| 担当A | Unit 1（認証・社員管理） | Unit 3（有給休暇） |
| 担当B | Unit 2（打刻・勤怠一覧） | Unit 4（カレンダー・アラート） |

- 担当A: 認証系のセキュリティ設定 → それに依存する有給ワークフロー
- 担当B: ドメインロジック（勤務時間計算） → それに依存する残業アラート
